# AskU — Evaluation Plan (MVP)

## Purpose
Evaluate whether AskU reduces the barrier to asking course-related questions while maintaining safety via verified anonymity and moderation.

## Research questions
RQ1: Does verified anonymity make students more willing to ask questions?
RQ2: Do users perceive AskU as safe/trustworthy due to moderation features?
RQ3: Is the platform usable for basic Q&A tasks in a course context?

## Participants
- Target: 8–12 University of Tartu students (mix of local + international if possible).
- Recruitment: classmates, course group chats, friends from other courses.
- Inclusion: enrolled students who have experience with Moodle/Telegram/Discord course chats.

## Study format
- Remote or in-person, 25–30 minutes per participant.
- Data collected: task completion, short questionnaire (SUS), short interview.

## Tasks (script)
T1: Open AskU, select a course, find the feed.
T2: Create a new question anonymously (course-scoped pseudonym).
T3: Open a thread and post a comment/answer.
T4: Report a toxic/spam comment using the report flow.

## Metrics
### Usability
- Task success rate (%) for T1–T4
- Time-on-task (approx.) per task
- Errors/confusions observed

### Perception & acceptance
- SUS score (System Usability Scale)
- 3 short Likert questions (1–5):
  Q1: “I would feel comfortable asking questions here.”
  Q2: “I feel this platform is safe from abuse.”
  Q3: “I prefer this over current course channels.”

### Engagement proxy (if pilot is available)
- #threads per week per course
- #comments per thread
- time-to-first-answer (median)
- #reports and moderation actions

## Questionnaire
### SUS (standard 10 items)
Use the standard SUS form (1–5 scale).

### Additional questions (3)
1) What would stop you from using AskU?
2) Which feature is most important for feeling safe?
3) What is missing compared to Moodle/Telegram?

## Data handling & ethics
- Participation is voluntary; participants can stop anytime.
- Do not collect names/student IDs in the results document.
- Store notes and survey answers without direct identifiers.
- Log only aggregated usage metrics for analysis.

## Analysis plan (simple)
- Report descriptive stats: averages/medians, % success rates.
- Summarize qualitative feedback into 3–5 themes.
- Connect findings to RQ1–RQ3.

## Deliverables (outputs)
- Table of results (tasks + SUS + key comments)
- Short summary of findings + limitations
- List of changes to implement after feedback (if time allows)
